## What is Windows-MCP?

Windows-MCP is a lightweight, open-source MCP (Model Context Protocol) server that enables seamless integration between AI agents and the Windows operating system. It acts as a bridge between Large Language Models (LLMs) and Windows, allowing agents to perform real desktop automation tasks.

## Overview

Windows-MCP provides **17 powerful tools** that give AI agents the ability to:

- **Interact with Windows UI** - Click buttons, type text, scroll, and navigate applications
- **Manage files and folders** - Read, write, copy, move, delete, and search files
- **Control applications** - Launch apps, resize windows, switch between programs
- **Execute system commands** - Run PowerShell scripts with full system access
- **Automate browsers** - Extract web content using DOM mode for clean data
- **Monitor processes** - List, filter, and terminate running applications
- **Access system resources** - Clipboard, registry, notifications, and more

## How It Works

1. **MCP Server** - Windows-MCP runs as an MCP server on your Windows machine
2. **Tool Registration** - It exposes 17 tools via the Model Context Protocol
3. **LLM Integration** - Your MCP client (Claude Desktop, Gemini CLI, etc.) connects to the server
4. **AI-Driven Automation** - The LLM calls tools to perform actions on your behalf

## Architecture

Windows-MCP follows a layered service architecture:

- **Desktop Service** - High-level orchestrator for window operations, screenshots, and input
- **Tree Service** - Captures Windows accessibility tree to identify interactive UI elements
- **UIAutomation Wrapper** - Low-level abstraction over Windows UIAutomation COM API
- **WatchDog Service** - Background thread monitoring UI focus changes in real-time
- **Analytics Service** - Optional PostHog telemetry (disabled with `ANONYMIZED_TELEMETRY=false`)

## System Requirements

<Note>
  **Operating Systems**: Windows 7, 8, 8.1, 10, or 11
</Note>

**Prerequisites:**
- Python 3.13 or higher
- UV package manager (`pip install uv` or `curl -LsSf https://astral.sh/uv/install.sh | sh`)
- English as default Windows language (recommended for App tool compatibility)

## Performance

Windows-MCP is designed for real-time interaction:

- **Action Latency**: 0.2-0.9 seconds between actions (click to click, type to type)
- **Screenshot Resolution**: Capped at 1920x1080 for token efficiency
- **UI Element Fetching**: Retry logic with 3 max retries for reliability
- **Multi-Threaded**: ThreadPoolExecutor for parallel UI tree traversal

<Tip>
  Latency varies based on system load, number of active applications, and LLM inference speed.
</Tip>
