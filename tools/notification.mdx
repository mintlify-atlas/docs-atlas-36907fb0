The Notification tool sends native Windows toast notifications, useful for alerting users about important events, task completion, or status updates when operating the desktop remotely.

## Parameters

<ParamField path="title" type="string" required>
  The notification title. Displayed prominently at the top of the toast notification.
  
  Keep concise (typically 1-5 words) for best readability.
</ParamField>

<ParamField path="message" type="string" required>
  The notification message body. Contains the detailed information you want to communicate.
  
  Can be longer than the title but should remain brief for better user experience.
</ParamField>

## Return Value

Returns a string confirming the notification was sent, or an error message if the operation failed.

## Usage Examples

### Simple Task Completion Alert

```python
# Notify user when a task completes
notification_tool(
    title="Task Complete",
    message="Your report has been generated successfully."
)
```

### Status Update Notification

```python

    title="Backup Complete",
    message="All files have been backed up to the cloud."
)
```

### Error or Warning Alert

```python

    title="Action Required",

```

### Progress Notification


## Best Practices

<Tip>
  Use notifications for important events that require user attention, especially when automating tasks that run in the background or when you're controlling a remote desktop.
</Tip>

<Note>
  Windows toast notifications appear in the Action Center and disappear automatically after a few seconds unless the user interacts with them.
</Note>

<Warning>
  Avoid sending too many notifications in quick succession, as this can be disruptive to the user and may cause notification fatigue.
</Warning>

## Common Use Cases

1. **Remote Desktop Automation**: Alert the user when automated tasks complete on a remote machine
2. **Long-Running Tasks**: Notify when processes like downloads, backups, or data processing finish
3. **User Interaction Required**: Alert when automation encounters a situation requiring human decision
4. **Status Updates**: Provide progress updates for multi-step workflows
5. **Error Handling**: Inform users when automated processes encounter errors

## When to Use Notifications

### Good Use Cases

- **Task completion**: "File conversion complete"
- **User input needed**: "Please approve the next step"
- **Important events**: "Security scan detected issues"
- **Background process updates**: "Database backup finished"

### Avoid Notifications For

- **Routine operations**: Don't notify for every small step in a workflow
- **Expected behavior**: Avoid notifying about normal, anticipated actions
- **High-frequency events**: Don't spam users with continuous notifications

## Technical Details

- Sends native Windows 10/11 toast notifications via the Desktop service
- Notifications appear in the Windows Action Center
- Toast notifications auto-dismiss after a system-defined timeout (typically 5-7 seconds)
- Notifications are non-blocking and don't interrupt user workflows

## Notification Behavior

- **Display**: Toast appears in the bottom-right corner of the screen
- **Sound**: Plays default Windows notification sound (system-dependent)
- **Action Center**: Notifications are logged in the Windows Action Center for later review
- **Focus Assist**: Respects Windows Focus Assist settings (may be suppressed during certain modes)
