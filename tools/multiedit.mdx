title: MultiEdit

The MultiEdit tool allows you to fill multiple text input fields at once, streamlining form filling and batch text entry operations. You can specify targets either by coordinates or by UI element labels from a Snapshot.

## Parameters

<ParamField path="locs" type="array">
  Array of locations with text: `[[x1, y1, text1], [x2, y2, text2], ...]`
  
  Each entry is a three-element array:
  - `x`: X coordinate of the input field
  - `y`: Y coordinate of the input field
  - `text`: Text to enter into the field
  
  Provide either `locs` or `labels`.
</ParamField>

<ParamField path="labels" type="array">
  Array of label-text pairs: `[[label1, text1], [label2, text2], ...]`
  
  Each entry is a two-element array:
  - `label`: UI element label/ID from a Snapshot
  - `text`: Text to enter into the field
  
  Provide either `locs` or `labels`.
</ParamField>

## How It Works

The MultiEdit tool:
1. Iterates through each location or label
2. Clicks on the input field to focus it
3. Types the specified text into the field
4. Moves to the next field

This is significantly faster than manually clicking and typing into each field individually.

## Usage Examples

### Form Filling with Coordinates

```python
# Fill out a registration form using coordinates
multi_edit_tool(
    locs=[
        [300, 150, "John Doe"],        # Name field
        [300, 200, "john@example.com"], # Email field
        [300, 250, "555-1234"],         # Phone field
        [300, 300, "123 Main St"]       # Address field
    ]
)
```

### Form Filling with Labels

```python
# First, take a snapshot to get field labels
snapshot_result = state_tool(use_ui_tree=True)

# Then fill fields using their labels
multi_edit_tool(
    labels=[
        [42, "John Doe"],          # Name field label 42
        [43, "john@example.com"],  # Email field label 43
        [44, "555-1234"],          # Phone field label 44
        [45, "123 Main St"]        # Address field label 45
    ]
)
```

## Return Value

Returns a success message listing all edited fields:

```
Multi-edited elements at: (300,150) with text 'John Doe', (300,200) with text 'john@example.com', (300,250) with text '555-1234'
```

## Working with Snapshot

To use the `labels` parameter, you must first call the Snapshot tool:

```python
# Step 1: Take snapshot to get UI element labels
snapshot_result = state_tool(use_ui_tree=True)

# Step 2: Review snapshot output to identify input field labels
# Look for text fields, text boxes, and editable elements

# Step 3: Use labels in MultiEdit
multi_edit_tool(
    labels=[
        [15, "First text"],
        [16, "Second text"],
        [17, "Third text"]
    ]
)
```

## Error Handling

```python
# Neither locs nor labels provided
"ValueError: Either locs or labels must be provided."

# Desktop state not available when using labels
"ValueError: Desktop state is empty. Please call Snapshot first."

# Invalid label format
"ValueError: Each label item must be [label, text]. Invalid: [15]"

# Failed to find element
"ValueError: Failed to process label item [99, 'text']: <error details>"
```

## Coordinates Format

<Info>
**locs format**: `[[x, y, text], [x, y, text], ...]`

Each entry must have exactly **three elements**:
1. X coordinate (integer)
2. Y coordinate (integer)
3. Text to type (string)

Example:
```python
locs = [
    [100, 200, "Hello"],
    [100, 250, "World"]
]
```
</Info>

## Labels Format

<Info>
**labels format**: `[[label, text], [label, text], ...]`

Each entry must have exactly **two elements**:
1. Element label/ID from Snapshot (integer)
2. Text to type (string)

Example:
```python
labels = [
    [42, "Hello"],
    [43, "World"]
]
```
</Info>

## Best Practices

<Tip>
**Tips for effective multi-editing:**

1. **Use labels when possible**: Labels from Snapshot are more reliable than hardcoded coordinates
2. **Take fresh snapshots**: Always call Snapshot immediately before using labels to ensure accuracy
3. **Verify field order**: Check that your data matches the order of fields on the form
4. **Handle focus properly**: Ensure no overlapping windows or pop-ups interfere with field access
5. **Consider field validation**: Some fields may have input restrictions (e.g., numeric only, email format)
6. **Test with small batches**: Verify the tool works with 1-2 fields before filling large forms
</Tip>
