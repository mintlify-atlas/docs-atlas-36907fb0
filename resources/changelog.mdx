## Recent Updates

Windows-MCP is actively developed with regular updates and improvements. Here are the latest highlights:

### Latest Features

- **VM Support**: Windows-MCP now supports cloud-hosted Windows automation via [windowsmcp.io](https://windowsmcp.io)
- **2M+ Users**: Reached over 2 million users in [Claude Desktop Extensions](https://claude.ai/directory)
- **PyPI Distribution**: Now available on [PyPI](https://pypi.org/project/windows-mcp/) with `uvx windows-mcp` support
- **MCP Registry**: Added to the official [MCP Registry](https://github.com/modelcontextprotocol/registry)
- **Windows-Use Agent**: Try [Windows-Use](https://pypi.org/project/windows-use/), an agent built on Windows-MCP

### New Tools

- **Registry Tool**: Read, write, delete, or list Windows Registry values and keys
- **Notification Tool**: Send Windows toast notifications with title and message
- **Process Tool**: List running processes or terminate them by PID or name
- **MultiEdit Tool**: Enter text into multiple input fields at specified coordinates
- **MultiSelect Tool**: Select multiple items (files, folders, checkboxes) with optional Ctrl key

### Platform Support

Windows-MCP supports:
- Windows 7
- Windows 8, 8.1
- Windows 10
- Windows 11

## Upgrade Guide

### Upgrading from PyPI

If you installed via `uvx`:

```bash
uvx --reinstall windows-mcp
```

### Upgrading from Source

If you cloned the repository:

```bash
cd Windows-MCP
git pull origin main
uv sync
```

### Configuration Updates

Most updates are backward compatible. If configuration changes are required, they will be noted in the specific release notes on GitHub.

## Roadmap

### In Progress

- Improved text selection within paragraphs (currently limited by a11y tree)
- Enhanced Type-Tool for IDE programming workflows
- Better multi-language support

### Limitations

Some current limitations we're working on:
- **Text selection**: Selecting specific sections of text in a paragraph is limited by accessibility tree constraints
- **IDE typing**: Type-Tool types entire programs at once in files; optimizing for incremental code entry
- **Gaming**: This MCP server is not designed for video game automation
